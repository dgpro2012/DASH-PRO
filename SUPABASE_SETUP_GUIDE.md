# Guía Paso a Paso: Conexión con Supabase

Esta guía explica cómo migrar tus datos de Google Sheets a Supabase para mejorar la velocidad y estabilidad del Dashboard.

## 1. Configuración de Supabase

1.  Ve a [Supabase.com](https://supabase.com/) y crea una cuenta.
2.  Crea un **Nuevo Proyecto** (New Project).
    *   Dale un nombre (ej: `DashboardData`).
    *   Establece una contraseña segura para la base de datos.
    *   Selecciona una región cercana a ti (ej: US East).
3.  Una vez creado el proyecto, ve a **Project Settings** (icono de engranaje) -> **API**.
4.  Copia los siguientes valores, los necesitarás para el Dashboard:
    *   **Project URL** (URL del proyecto)
    *   **anon public** (API Key pública)

## 2. Creación de Tablas (Base de Datos)

Ve al **SQL Editor** (icono de terminal en la barra lateral izquierda) y pega el siguiente código para crear las tablas necesarias. Luego dale a "Run".

```sql
-- Tabla para Facebook Ads
CREATE TABLE facebook_ads (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    date DATE NOT NULL,
    campaign_name TEXT,
    adset_name TEXT,
    ad_name TEXT,
    spend NUMERIC,
    impressions INTEGER,
    clicks INTEGER,
    leads INTEGER, -- Messaging Conversations Started
    purchases INTEGER,
    currency TEXT DEFAULT 'USD',
    usd_real NUMERIC,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now())
);

-- Tabla para Kommo Leads
CREATE TABLE kommo_leads (
    lead_id BIGINT PRIMARY KEY, -- ID único del lead en Kommo
    lead_name TEXT,
    price NUMERIC,
    status_name TEXT,
    pipeline_id BIGINT,
    created_at TIMESTAMP WITH TIME ZONE,
    closed_at TIMESTAMP WITH TIME ZONE,
    country TEXT,
    product TEXT,
    source TEXT,
    keywords TEXT, -- Palabras Claves / PC
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now())
);

-- Tabla para Ventas Manuales
CREATE TABLE ventas_manuales (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    date DATE,
    amount NUMERIC,
    country TEXT,
    product TEXT,
    source TEXT,
    pc TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now())
);
```

## 3. Conexión de Datos (Facebook y Kommo -> Supabase)

El Dashboard solo **lee** datos. Para **enviar** datos desde Facebook y Kommo a Supabase, necesitas una herramienta de automatización como **Make.com** (antes Integromat) o **Zapier**.

### Opción A: Usando Make.com (Recomendado)

#### Para Facebook Ads:
1.  Crea un nuevo escenario.
2.  **Trigger:** Facebook Insights -> "Get Insights".
    *   Configura para que corra cada día (ej: a las 6:00 AM).
    *   Selecciona tu cuenta publicitaria.
    *   Nivel: Ad (Anuncio).
    *   Campos: Date, Campaign Name, Ad Set Name, Ad Name, Spend, Impressions, Clicks, Actions (Purchases, Messaging Conversations).
3.  **Action:** Supabase -> "Upsert a Row" (Insertar o Actualizar).
    *   Tabla: `facebook_ads`.
    *   Mapea los campos de Facebook a las columnas de Supabase.
    *   **Clave Única:** Para evitar duplicados, puedes crear un ID compuesto en Make (ej: `Date` + `Ad ID`) o borrar los datos del día antes de insertar los nuevos.

#### Para Kommo (Leads):
1.  Crea un escenario.
2.  **Trigger:** Kommo (o Webhook) -> "Watch Leads".
3.  **Action:** Supabase -> "Upsert a Row".
    *   Tabla: `kommo_leads`.
    *   Mapea: `Lead ID` -> `lead_id`, `Price` -> `price`, `Created At` -> `created_at`, etc.
    *   Usa `lead_id` como la clave para actualizar si el lead cambia de estado.

## 4. Conexión con el Dashboard

1.  Abre tu Dashboard.
2.  Ve al panel de **Configuración** (icono de engranaje).
3.  En la sección de la marca (PECAS o LAS MEJORES):
    *   Cambia "Fuente de Datos" a **SUPABASE**.
    *   Pega la **Supabase URL** que copiaste en el paso 1.
    *   Pega la **Supabase Key** (anon public) que copiaste en el paso 1.
    *   (Opcional) Si cambiaste los nombres de las tablas, escríbelos en los campos correspondientes. Si usaste los del código SQL de arriba, déjalos vacíos o usa los por defecto (`facebook_ads`, `kommo_leads`).
4.  Guarda la configuración.

¡Listo! El Dashboard ahora leerá los datos directamente de Supabase, lo que debería ser mucho más rápido y estable que Google Sheets.
